New Specification Notice

The Open Agent Format is a newly released specification. While we're working with major vendors to adopt this standard, most platforms haven't fully integrated OAF yet. We provide vendor-specific mapping guides below to help you convert between OAF and existing formats.

What is Open Agent Format?

The Open Agent Format (OAF) is an open, interoperable specification for defining AI agents with their instructions, compositions, and configurations. OAF enables you to create agents once and deploy them across multiple platforms including Claude Code, Goose, Deep Agents, Letta, and others.

Think of OAF as a universal language for AI agents - write your agent definition in OAF format, and it can work anywhere.

Core Design Principles

Filesystem as Truth

The directory structure and files define the agent - no hidden state or configuration

Interoperable

Works across Claude Code, Goose, Deep Agents, Letta, and other harnesses

Composable

Agents reference skills, packs, weblets, MCPs, and other agents

Version-aware

Built-in semantic versioning and version history support

Human-readable

Markdown for instructions, YAML for metadata - easy to read and edit

Harness-agnostic

Core format is independent of any specific platform

Key Features

  • Single Manifest File: All agent configuration in one AGENTS.md file
  • Skills Integration: Compatible with AgentSkills.io specification
  • MCP Support: Built-in Model Context Protocol server configuration
  • Sub-Agent Delegation: Compose complex agents from simpler specialized agents
  • Tool Subset Selection: Avoid context overflow with selective tool access via ActiveMCP.json
  • Compliant with Standards: Built on AGENTS.md and AgentSkills.io specifications

Quick Example

A minimal OAF agent requires just one file:

my-agent/
└── AGENTS.md

The AGENTS.md file contains YAML frontmatter with metadata and Markdown instructions:

---
name: "My Assistant"
vendorKey: "acme"
agentKey: "assistant"
version: "1.0.0"
slug: "acme/assistant"
description: "A helpful AI assistant"
author: "@acme"
license: "MIT"
tags: ["assistant", "general"]
---

# Agent Purpose
I am a helpful AI assistant that can answer questions and help with tasks...

Getting Started

Ready to create your first OAF agent? Here's what you need to know:

  1. Create a directory for your agent (e.g., my-vendor/my-agent/)
  2. Add an AGENTS.md file with metadata and instructions
  3. Optionally add skills, MCP configs, or sub-agents as needed
  4. Deploy to your preferred harness using the mapping guides above

For detailed specifications, examples, and best practices, see the full documentation.
